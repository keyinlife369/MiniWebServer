一、项目简介

MiniWebServer 是一套完整Windows平台C++高性能Web服务框架，采用异步IO多线程架构，搭配Qt可视化管理界面，集成静态资源托管、RESTful接口、MySQL持久化、HTTPS加密、实时连接监控、日志系统等全套能力。

适用场景：小型后台接口服务、本地管理面板、留言板业务演示、C++网络编程学习工程。

二、完整技术栈

底层网络 & 语言

  • 标准：C++20，兼容现代STL特性（string_view、atomic、shared_mutex等）
  • 异步IO：Boost.Asio 跨平台异步网络库，替代原生epoll/iocp
  • 多线程：自定义通用线程池，处理业务请求解耦IO线程

上层功能依赖

  • GUI界面：Qt6 Widget，服务器可视化控制面板
  • 数据库：MySQL9.5 C原生API，自研RAII连接池
  • 加密传输：OpenSSL，实现HTTPS TLS1.2+双端口监听
  • 单元测试：GoogleTest(GTest)，48个全覆盖测试用例

构建 & 部署工具

  • 编译工具：CMake 3.16+，支持Ninja构建
  • 容器：Docker多阶段构建 + docker-compose MySQL编排
  • 环境：Windows VS2022 / Ubuntu Linux 双平台编译

五、HTTP协议解析核心能力

1. 流式分段解析

不再要求一次性接收完整请求，缓冲区增量追加Body，网络分片分包也能正常解析，解决大请求/慢速网络丢包问题。

2. HTTP流水线支持

解析完成后不清空缓冲区，保留下一条请求残留数据，单连接连续处理多条请求，大幅提升吞吐。

3. 容错与限制

  • 非法Content-Length捕获异常，返回400
  • Header自动修剪空格、大小写兼容Keep-Alive识别
  • 可配置最大请求体，超大请求直接拒绝
  • 不支持的HTTP方法返回405 Method Not Allowed

六、路由与REST API系统

1. 两种路由匹配模式

  • 精确路由：如 GET /api/info，存入unordered_map O(1)极速查找
  • 参数路由：如 DELETE /api/message/:id，自动捕获路径参数存入HttpRequest.route_params

2. 响应封装 HttpResponse

// 内置静态工厂方法
HttpResponse::ok(body)        // 200正常返回
HttpResponse::created(body)   // 201创建成功
HttpResponse::badRequest(msg) // 400参数错误
HttpResponse::notFound(msg)   // 404路径不存在
HttpResponse::serverError()   // 500服务异常

3. 内置演示API

  1. GET /api/info：获取服务器运行统计、连接数、QPS
  2. POST /api/echo：回显表单/请求体，用于POST调试
  3. POST /api/messages：新增留言入库
  4. GET /api/messages：批量查询所有留言
  5. GET /api/message/:id：单条留言查询
  6. DELETE /api/message/:id：删除指定留言

七、MySQL数据库连接池

架构设计

分为两层：MysqlConnection单连接RAII封装、MysqlPool全局连接池。

  • 程序启动预创建N条数据库连接（默认8）
  • 多线程并发获取连接时条件变量阻塞等待空闲连接
  • 归还连接自动ping检测失效，失效自动重连/重建
  • RAII守卫自动归还连接，无需手动释放，无泄漏

安全机制

所有用户输入调用 escape() 使用mysql_real_escape_string转义字符串，彻底杜绝SQL注入漏洞。

自动初始化

GUI点击启动服务器时，自动执行建库、建表SQL，开箱即用无需手动执行脚本。

八、HTTPS TLS加密支持

多态连接设计

基类Connection管理通用socket逻辑，派生SslConnection封装ssl::stream SSL流，统一接口，读写函数自动区分HTTP/SSL连接。

功能特性

  • 独立SSL监听端口8443，与HTTP 8080同时运行
  • 禁用老旧SSLv2/SSLv3，仅使用现代加密套件
  • 自签名RSA4096证书，有效期一年，存放certs目录
  • HTTPS响应自动添加HSTS强制加密头部
  • 完整SSL握手、读写、关闭流程错误捕获

九、Qt可视化监控面板

控制面板区域

  • 端口、工作线程数、静态目录自定义配置
  • 一键启动/停止服务器按钮
  • 运行中锁定配置，防止动态修改冲突

实时统计卡片

  • 活动连接数量、总请求数、每秒QPS
  • 上下行带宽自动格式化B/KB/MB/s
  • 服务器运行时分秒计时展示

活动连接表格

实时刷新所有TCP连接信息：客户端IP、端口、连接建立时间、最后活跃时间、该连接处理请求数、发送总字节，方便排查长连接占用。

彩色日志窗口

分级彩色日志，自动时间戳，支持清空，日志同时写入server.log文件持久保存，最大限制1000行避免内存溢出。

十一、Docker容器部署

Dockerfile多阶段构建

  • 第一阶段builder：完整编译环境，编译服务、压测工具
  • 第二阶段runtime：仅运行时依赖，镜像体积大幅缩小

docker-compose 编排

一键拉起MySQL9.5容器，自动执行初始化SQL创建留言表，数据卷持久化存储留言数据，内置健康检查监控数据库状态。

十二、编译构建说明 CMake

依赖配置方式

  • Boost、MySQL、OpenSSL支持环境变量/CMake参数指定路径
  • Windows自动复制所有运行依赖dll至输出目录
  • 自动拷贝静态网页、TLS证书文件夹到程序同级目录

编译可选开关

  • -DBUILD_TESTS=ON/OFF 是否编译单元测试
  • -DBUILD_BENCH=ON/OFF 是否编译压测工具

输出产物

  1. MiniWebServer：主GUI服务程序
  2. MiniWebServer_tests：单元测试可执行文件
  3. MiniWebServer_bench：并发压测客户端

十三、内置前端演示页面

  1. index.html 首页：项目介绍、功能导航链接
  2. form.html POST表单测试页：提交表单调用/api/echo调试接口
